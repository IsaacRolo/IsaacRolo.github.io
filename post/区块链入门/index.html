区块链入门

2021年4月12日

区块链的本质

区块链是一种特殊的分布式数据库

  • 区块链的主要作用是储存信息。任何需要保存的信息，都可以写入区块链，也可以从里面读取，所以它是数据库。
  • 任何人都可以架设服务器，加入区块链网络，成为一个节点。区块链的世界里面，没有中心节点，每个节点都是平等的，都保存着整个数据库。你可以向任何一个节点，写入/读取数据，因为所有节点最后都会同步，保证区块链一致。

区块链特点

  • 去中心化：所有节点最后都会同步，保证区块链一致
  • 数据可信，防篡改：区块头存储上一个区块的Hash

概念

  • 节点：区块链网络中的一个节点，每个节点都保存了所有的区块链

  • 区块

    • 区块头

      包含当前区块的多项特征值

      • 生成时间
      • 实际数据（即区块体）的哈希
      • 上一个区块的哈希
      • …

    • 区块体

  • Hash：计算机可以对任意内容，计算出一个长度相同的特征值，只要原始内容不同，对应的哈希一定（概率极低）不同

  • 区块的不可修改性

    • 修改当前区块 -> 当前区块的Hash会改变 -> 链上的下一个区块找不到当前区块（区块头存放上一个区块的哈希） -> 当前区块脱离链

    • 同时修改多个区块的内容不可能发生，除非有人掌握了全网51%以上的计算能力

  • 新增区块的速度

    设计者故意让添加新区块，变得困难，为了缓解添加区块和同步区块之间的冲突

  • 计算Hash的难度系数

    只有满足条件的哈希才会被区块链接受。这个条件特别苛刻，使得绝大部分哈希都不满足要求，必须重算。

  • 区块链的分叉

    • 两个人同时向区块链写入数据，同时有两个区块加入，因为它们都连着前一个区块，就形成了分叉
    • 规则：新节点总是采用最长的那条区块链，拥有大多数计算能力的那条分支，就是正宗的区块链

场景

将数据（文件）存到区块链上，达到防篡改、可追溯及数据来源可信任的目的

区块链取证采用文件与哈希值分离的方式，链上只保存文件的哈希值，原文件保存在链下。可实现对证据真实性的快速核验

存证（上链）

将文件或结构化数据的哈希值存在区块链上，未来可对该文件进行核验。保证数据不可篡改性，一旦篡改将会暴露。
